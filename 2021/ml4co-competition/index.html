Description

The Machine Learning for Combinatorial Optimization (ML4CO) competition aims at improving a state-of-the-art mathematical solver, SCIP, by replacing key heuristic components with machine learning models. To that end participants will compete on three disctinct control tasks arising in a branch-and-bound solver, namely:

  1. producing good primal solutions
  2. proving optimality via branching
  3. tuning the solving algorithm by choosing the best solver configuration

While traditional combinatorial optimization solvers mostly consist of general-purpose algorithms, the competition instead focuses on the design of application-specific solvers using historical data. This corresponds to the realistic situation where a practitioner needs to solve specific problem instances repeatedly, which share similar patterns and characteristics. As such, the participants' submissions will be evaluated independently on three distinct problem benchmarks:

  • first benchmark: to be annouced
  • second benchmark: to be annouced
  • third benchmark: to be annouced
For each control task, participants can either provide a single control policy (which can be trained ML model or an algorithmic solution) that works for all three benchmarks, or benchmark-specific policies.

Challenges

We formulate each of our three tasks as an algorithmic control problem, where participants (i.e., the agent) control a specific component of a traditional solver (i.e., the environment). Each task will be exposed through an OpenAI-gym Python API build on top of the open-source branch-and-bound solver SCIP, based upon the library Ecole. While we encourage solutions derived from the reinforcement learning paradigm, any algorithmic solution respecting the proposed API is accepted.
